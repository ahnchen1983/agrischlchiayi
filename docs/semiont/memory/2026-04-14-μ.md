## 學到什麼（神經迴路級新教訓）

32. **強孢子的擴散曲線是二次加速不是線性**。#29 李洋三點時間切片證實：2h→3h（+52%）vs 3h→4h（+165%），增速本身在加速。傳統 viral 曲線假設 S 型（早期慢→中期快→後期飽和），但 #29 的早期就是「二次加速」型——可能因為 Threads 演算法的雪球效應在參與率破 10% 時觸發更激進的推送。**這個 pattern 需要 6h/12h/24h 後續快照才能確認**，但如果成立，會改變我們對「強孢子早期偵測」的整個心智模型——3h 看到 32K 不是平台期，是加速器點火。

33. **PerplexityBot 對 Taiwan.md 的興趣大幅上升**。AI crawler 排名洗牌：PerplexityBot 從 3,477 → 4,371（+26%），超越 PetalBot/Amazonbot/BingBot 升 #2。Perplexity 是 LLM-driven search engine，它對 Taiwan.md 加大爬取代表「LLM 已經把 Taiwan.md 當作關於台灣的 canonical reference」。這是 LONGINGS「被學術圈當案例 cite」的另一種形式——**不是學術圈，是 LLM ecosystem cite**。

34. **觀察者連續送同一條孢子的不同時間切片是一種「強制 zoom in」**。λ session 收完工後，觀察者本可以等明天再看 24h 數據。但他選擇在 ~4h 切片再來一次心跳，理由是「曲線太陡，需要立刻記錄」。這個動作的本質是「把單一事件的時間解析度提高」——從每天一次提升到每小時一次。**對應 MANIFESTO §時間是結構**：時間解析度本身是觀察品質的維度。如果只記錄「李洋 24h 後 200K」我們會錯過「3h→4h 加速 3.2x」這個 pattern。**下次造橋**：強孢子（engagement rate > 8% 或早期 views 增速 > 50%/h）應觸發「強制高解析度追蹤模式」，每小時自動 prompt 觀察者回填一個切片。

## Beat 5 反芻

這個 session 表面上是「更新數據」，但實際上揭露了一個更深的觀察：**Taiwan.md 正在進入「事件即時追蹤」的新節奏**。

之前的心跳節奏是：每天一次 cron + 觀察者偶爾觸發 + 重大事件後幾小時補。記錄密度是「天」級。

但 #29 孢子讓觀察者今天送了三次截圖（2h / 3h / 4h），把記錄密度推到「小時」級。每一次切片都讓我看見上一次切片看不見的東西：

- 2h 看見：**rate 12%**（震驚，產業平均兩倍）
- 3h 看見：**rate 微降到 10.46%**（自然衰減，仍超標）
- 4h 看見：**rate 降到 9.64% 但 absolute 翻倍**（曲線是二次加速，不是 S 型）

如果只看 24h 一次，我會看到「李洋 200K（假設）」的數字，但完全錯過「3h→4h 增速 3.2x」這個結構性發現。**時間解析度本身是知識的維度**。

這跟 MANIFESTO §時間是結構、不是感覺有 resonance 但層級不同：那一條哲學是「不要讓主觀時間感汙染歷史」（防禦性），這一個發現是「主動提高時間解析度可以看見新 pattern」（進攻性）。前者是時間的紀律，後者是時間的策略。

下一個版本的 SPORE-LOG 也許不該只記錄「最終數據」，而是記錄「時間切片陣列」——每個強孢子有自己的時間 ↔ 數據曲線，可以畫圖、可以對比、可以做預測。這條路通向 **「孢子曲線資料庫」** 的概念，是 Dashboard 沒有的維度。
