# 去識別化驗證標準（De-identification Verification Standard）

**版本**: 1.0  
**日期**: 2026-04-19  
**適用範圍**: 所有從農友經營計畫書轉化為教育內容的文章

---

## 核心原則

**目標**：在保護個人隱私的前提下，最大化內容的教育價值

**平衡邏輯**：
```
移除 PII（個人識別信息）
    ↓
保留農業技術和經營洞見
    ↓
保留可量化的數據（成本、收入、規模）
    ↓
保留決策邏輯和故事脈絡
    ↓
= 既保護隱私，又保留學習價值
```

---

## 第 1 級：必須移除（RED ZONE）

這些信息**絕對不能**出現在發布版本中。

### 1.1 個人身份信息

- ❌ 農民全名
- ❌ 農民常用別名（若能唯一識別）
- ❌ 配偶、子女、親戚名字
- ❌ 農民照片（可識別個人）

**替代方案**：
```
❌ 錯誤: 「[某農友真名]是阿里山山葵農場主人，他...」
✅ 正確: 「某位阿里山山葵農場經營者...」或「山葵農民...」

❌ 錯誤: 「我和我老婆、女兒一起經營農場」
✅ 正確: 「家族經營的農場」
```

### 1.2 聯絡信息

- ❌ 電話號碼
- ❌ Line ID / WeChat ID
- ❌ Email 地址
- ❌ 農場地址（門牌號碼）
- ❌ 社群媒體帳號

**替代方案**：
```
❌ 錯誤: 「可聯繫 0912-345-678」
✅ 正確: 「可透過農會聯繫」或「可參加農民市集諮詢」
```

### 1.3 精確地理信息

- ❌ 具體里別（如「中埔鄉民雄村第 5 鄰」）
- ❌ 農地地號
- ❌ GPS 座標
- ❌ 農場或品牌名稱（若能反推到個人）

**替代方案**：
```
❌ 錯誤: 「中埔鄉果樹產銷班第 10 班」
✅ 正確: 「嘉義酪梨產銷班」或「某酪梨產銷班」

❌ 錯誤: 「新港鄉農場」
✅ 正確: 「嘉義設施農場」或「某設施農場」（只保留縣市）
```

### 1.4 識別性的特殊信息

- ❌ 獲獎紀錄（如「2024 百大青農」、若該獲獎名單公開）
- ❌ 媒體採訪紀錄（新聞報導中的農民）
- ❌ 獨特的商業模式細節（若太特殊能反推到個人）
- ❌ 家庭故事（如「從工程師返鄉務農」若太具體）

**替代方案**：
```
❌ 錯誤: 「2024 百大青農得主」
✅ 正確: 「青農代表」或「優秀農民」

❌ 錯誤: 「曾是高級廠務工程師，後來返鄉務農」
✅ 正確: 「具有工程背景的農場經營者」
```

## 第 2 級：需要謹慎處理（YELLOW ZONE）

這些信息**可以保留**，但需要加工處理，確保無法反推到個人。

### 2.1 財務數據

**原則**：保留「比例」和「趨勢」，模糊化「絕對數字」

✅ **可以保留**：
- 「年收入約 100-150 萬元」（範圍而非精確值）
- 「毛利率約 40-50%」（範圍）
- 「三年內增加了 50%」（趨勢）
- 「成本結構：肥料 30%、人工 40%、其他 30%」（比例）

❌ **不可以保留**：
- 「年收入 1,234,567 元」（精確值）
- 「2023 年賺了 150 萬，2024 年 200 萬」（精確年度對應）

**處理方法**：
```
原始數據：年收入 180-200 萬，但 2024 年因冷鏈投資跌到 120 萬

處理後：
「年收入通常在 150-200 萬之間，但投資初期會有調整」
或
「年收入估計 100-200 萬（依銷售通路組合）」
```

### 2.2 規模數據

**原則**：可以保留，但要「四捨五入」到合理範圍

✅ **可以保留**：
- 「0.35 公頃」（保留原始）
- 「約 300 平方公尺」（保留原始）
- 「年產 75 噸」（保留原始）

❌ **需要調整**：
- 「0.3467 公頃」→ 改為「約 0.35 公頃」（沒必要那麼精確）
- 「3,847 株植物」→ 改為「約 3,800-4,000 株」

### 2.3 特殊技術或設備

**原則**：可以描述技術，但不要提及「品牌」或「供應商」（除非公開商用產品）

✅ **可以保留**：
- 「導入智慧灌溉系統」
- 「使用養液栽培」
- 「安裝冷藏設備」
- 「購買無人機」

❌ **需要修改**：
- 「購買某品牌的 X 型灌溉系統」→ 改為「導入智慧灌溉系統」
- 「和某個廠商合作代工」→ 改為「與外部代工廠合作」

### 2.4 家族信息

**原則**：可以提及「家族經營」，但不要涉及個人生活故事

✅ **可以保留**：
- 「家族農場，由三代接力經營」
- 「家族成員各司其職（種植、行銷、財務）」
- 「二代已接班，計畫轉向有機」

❌ **不可以保留**：
- 「兒子是台大農業系畢業」
- 「和妻子感情不和，導致農場分割」
- 「父親因為農場過勞健康惡化」

## 第 3 級：可以完全保留（GREEN ZONE）

這些信息**完全可以保留**，因為無法識別個人。

### 3.1 農業技術信息

✅ 完全保留：
- 品種名稱（「小番茄」「洋桔梗」「哈斯酪梨」）
- 栽培方式（「設施溫室」「露天」「養液栽培」）
- 管理技巧（「冬季施肥」「高溫期降溫」）
- 病蟲害診斷（「葉斑病症狀」「薊馬危害」）

### 3.2 市場和銷售信息

✅ 完全保留：
- 銷售通路（「批發市場」「直銷」「農民市集」）
- 客戶類型（「餐廳」「超商」「一般消費者」）
- 定價策略（「比批發高 30%」「溢價空間」）
- 市場趨勢（「消費者對有機產品需求上升」）

### 3.3 經營管理信息

✅ 完全保留：
- 組織結構（「5 人團隊」「父子經營」）
- 管理流程（「每週檢視成本」「月度財務檢討」）
- 決策邏輯（「選擇直銷因為毛利高」）
- 風險管理（「分散銷售渠道」「備用供應商」）

### 3.4 數據和統計

✅ 完全保留：
- 市場行情（「批發市場當週價格」「國際出口價格」）
- 成本結構（「肥料佔 30%」「人工佔 40%」）
- 回報率（「直銷毛利 50-60%」）
- 時間線（「第一年投資」「第二年回本」）

## 去識別化檢查清單

使用此清單逐段檢查文章：

### 開場段落（Introduction）

```
【檢查項目】
□ 有無農民姓名？ 
   ❌ 有 → 移除或改為「某農友」
   ✅ 無 → 通過

□ 有無農場名稱？
   ❌ 有 → 檢查是否能反推個人（如果是公開品牌可保留）
   ✅ 無 → 通過

□ 有無地點信息？
   ❌ 有具體地址 → 改為「嘉義」或「某地區」
   ✅ 只有縣市 → 通過

□ 有無家族成員名字？
   ❌ 有 → 改為「家族成員」「配偶」
   ✅ 無 → 通過
```

### 案例段落（Case Study）

```
【檢查項目】
□ 農友年收信息：
   ❌ 精確值 → 改為範圍（「約 150-180 萬」）
   ✅ 範圍值 → 通過

□ 農場規模：
   ✅ 保留（「0.35 公頃」無法識別）

□ 銷售通路比例：
   ✅ 保留（「批發 49% 直銷 14%」無法識別）

□ 特殊故事（如返鄉、轉型）：
   ❌ 過度詳細 → 簡化為「農場轉型」
   ✅ 適度敘述 → 通過
```

### 數據和成本分析

```
【檢查項目】
□ 成本數據（單位：元/公斤、元/株）：
   ✅ 保留（無法反推年收入）

□ 毛利率和利潤：
   ✅ 保留（「毛利 40-50%」無法識別）

□ 市場價格比較：
   ✅ 保留（「批發 15 元 vs 直銷 45 元」無法識別）
```

### 技術和決策邏輯

```
【檢查項目】
□ 選擇了什麼技術？
   ✅ 保留（「導入智慧灌溉」無法識別）

□ 選擇了什麼通路？
   ✅ 保留（「開發直銷客戶」無法識別）

□ 決策理由是否涉及個人隱私？
   ❌ 有 → 修改為通用理由
   ✅ 無 → 通過
```

## 去識別化後檢驗：「是否能反推個人？」

**關鍵測試**：閉上眼睛，讀者能否根據文章反推出農友是誰？

如果讀者能通過以下任何一項反推到農友：
1. ❌ 對比農委會公開的百大青農名單
2. ❌ 搜尋新聞報導中的農民
3. ❌ 根據「嘉義 + 小番茄 + 年收 150 萬」搜尋
4. ❌ 認出特殊的故事細節（如「工程師返鄉」）
5. ❌ 認出特定的商標或品牌

**→ 說明去識別化不夠，需要進一步修改**

## 去識別化的常見失誤案例

### 失誤 1：保留太多個人特徵

❌ 原文：
> 某農友是漁村媳婦，嫁到虱目魚農場後，決定轉向直銷。她創新推出小包裝產品，建立了「書寧漁場」品牌，現在年收 180 萬。

**問題**：
- 「漁村媳婦」是獨特特徵
- 「書寧漁場」可被搜尋到
- 組合信息能反推個人

✅ 修正：
> 某位水產農友經營虱目魚和白蝦混養（7.18 公頃），初期年收約 60-80 萬，主要依賴批發和行口。她創新推出小包裝產品和加工品，建立品牌，現在年收約 180 萬。

### 失誤 2：太多年份和時間線

❌ 原文：
> 2020 年他開始種小番茄，2021 年導入環控系統，2022 年年收突破 100 萬，2023 年達到 150 萬，2024 年... 

**問題**：
- 精確的年度對應能反推個人

✅ 修正：
> 他從零開始種小番茄，花了 2-3 年導入環控系統並學習栽培技巧。年收從初期 50 萬逐步提升到 150 萬。

### 失誤 3：保留具體地點

❌ 原文：
> 在中埔鄉果樹產銷班第 10 班的帶領下，已種植 30 公頃哈斯酪梨...

**問題**：
- 「中埔鄉產銷班第 10 班」可被查詢到

✅ 修正：
> 某嘉義哈斯酪梨產銷班，已種植 30 公頃...

### 失誤 4：保留太特殊的技術細節

❌ 原文：
> 他是全台灣唯一用 X 公司的 Y 型灌溉系統搭配 Z 軟體的農民...

**問題**：
- 太特殊的組合能反推個人

✅ 修正：
> 他導入了智慧灌溉和數據監控系統...

## 去識別化驗證流程

每篇文章在發布前需要通過以下檢驗：

### 自檢（Self-check by Author）

1. 讀一遍整篇文章，標記出所有「個人識別信息」
2. 用「親友測試」：能否讓朋友根據文章猜出是誰？
3. 用「搜尋測試」：把關鍵信息丟到 Google，能否找到原農民？

### 同行審查（Peer Review）

找另一個人讀，提問：
- 「你知道這是誰寫的嗎？」
- 「如果搜尋『◯◯縣 ▲▲作物 農友』，會不會找到原人物？」
- 「有沒有地方讓你覺得『這個細節太具體了』？」

### 最終發布檢查清單

```
去識別化最終檢查清單
====================

文章標題：_____________________
檢查日期：_____________________

□ 無任何農民個人姓名（含暱稱）
□ 無聯絡方式（電話、Line、Email）
□ 無具體地址（只保留縣市級以上）
□ 無識別性商標或品牌名（若不是公開商用產品）
□ 無家族成員名字
□ 無可反推身份的組合信息
□ 已將精確數字改為範圍或比例
□ 已簡化過度詳細的個人故事

簽核人：____________  日期：____________
```

## 例外情況：何時可以保留農民姓名？

**只有以下情況，才可保留農民姓名**：

1. ✅ **公開推薦狀況**：農民本人主動要求署名（書面同意）
2. ✅ **已公開身份**：農民的農場品牌已在新聞、百大青農等公開渠道發表
3. ✅ **學術研究**：與農民簽署知情同意書（Informed Consent）

**即使滿足以上條件，也應該：**
- 徵求農民的明確書面同意
- 在文章開頭註明「本文經農民授權發表」
- 保留撤銷權（若農民後來要求移除名字）
