## 📋 批量處理工作流

### 方法論：應用 FARMER-DATA-PROCESSING-CHECKLIST 6 步流程

對於每一篇新文章，遵循：

```
1️⃣ 基礎信息提取
   - 從 2-3 個農友案例中提取共同的作物、規模、問題
   
2️⃣ 分類對齐
   - 確認應放入哪個分類（按 CATEGORY-MAPPING-GUIDE）
   
3️⃣ 去識別化驗證
   - 移除農友姓名、地址、家庭信息
   - 保留：作物、規模、成本、技術、策略
   
4️⃣ 內容品質檢查
   - 標題清晰、有案例、有檢查清單、有 Wikilinks
   
5️⃣ 決定行動
   - 新文章（內容不重複）vs 補充現有文章
   
6️⃣ 最終發布
   - npm run sync → npm run build → git commit
```

## 🔄 工作流程示例

### 以「小黃瓜栽培技術」為例

**第 1 步：從農友數據中識別**
```
農友：Farmer-C、Farmer-D、Farmer-E（3 位）
共同點：都種設施小黃瓜 0.2-0.35 公頃
共同問題：產量穩定性、環控、病害防治
```

**第 2 步：提取信息**
- 作物特性：設施栽培，12-14 週採收，高水肥需求
- 規模參考：0.25 公頃為「適合小農的規模」
- 成本結構：肥料 5 萬、設施折舊 3 萬、人工 8 萬、銷售成本 3 萬 = 年成本約 20 萬
- 產量預期：75 噸/年（標準）
- 銷售通路：批發（50%）、行口（30%）、直銷（20%）

**第 3 步：去識別化**
```
原文：「Farmer-C在東石鄉種小黃瓜 0.35 公頃...」
改為：「某設施蔬果農民（小黃瓜 0.3 公頃，年產 75 噸）...」
```

**第 4 步：撰寫文章框架**
```
標題：小黃瓜設施栽培技術指南

Frontmatter:
- crop: ['小黃瓜']
- tech: ['設施農業', '環控系統', '滴灌']
- level: '進階'

章節：
1. 品種選擇（東石地區適合品種）
2. 設施規劃（0.25 公頃的標準配置）
3. 種苗與定植（時期、密度、活著率）
4. 肥水管理（參考多個農友的月份管理）
5. 病蟲害防治（常見問題）
6. 採收與分級
7. 成本分析（參考 3 位農友）
8. 銷售通路對比（批發 vs 直銷）

案例段落：
「某設施黃瓜農友（0.3 公頃，年產 75 噸），通過環控系統
和精準施肥，將單位面積年產從 250 公斤/平方米提升到
280 公斤/平方米，年收增加 15%...」
```

**第 5 步：發布**
```bash
git add knowledge/Crop-Production/小黃瓜設施栽培技術指南.md
npm run sync && npm run build
git commit -m "content: add cucumber hydroponic cultivation guide with case studies"
git push
```
