# 小農 IoT 灌溉導入指南 — 從零到自動化

## 導言

灌溉是設施農業的「生命線」。傳統灌溉依賴農民的「經驗眼光」——早上看土壤濕度，憑感覺決定澆多少水。這種做法不僅耗力（每天巡田），而且效率低下（產量波動大、病害易發、用水浪費）。

小農 IoT 灌溉的核心目標是：**用最少的投資，實現「自動化」——讓機器代替人工判斷，農民只需監控和調整策略**。

本指南基於嘉義多位農友的實戰經驗，提供從「0 元」到「30 萬」的逐步導入路徑。無論規模大小，都能找到適合的方案。

---

## 第 1 章：IoT 灌溉為什麼重要？

### 問題：傳統灌溉的三大痛點

#### 1. 耗時耗力
```
傳統做法：
  - 每天早晨巡田檢查土壤
  - 根據經驗判斷是否澆水
  - 手動調整水量（看「感覺」）
  - 全年 365 天，每天 30-60 分鐘

預估成本：
  - 1 個人 × 365 天 × 0.5-1 小時 = 年度 180-365 小時勞動力
  - 折合年成本：18-40 萬元（依工資而定）
```

#### 2. 產量不穩定
```
原因：灌溉時機和量都是「估計值」
  → 過多：根部爛、病害增加
  → 過少：植株缺水、採收期縮短
  
結果：同樣的設施、同樣的肥料，但產量波動 10-20%
      某年豐收，隔年減產，農民無法解釋原因
```

#### 3. 用水浪費
```
傳統灌溉損失：
  - 無法精準控制灌溉量 → 浪費 20-30% 水份
  - 表層土濕度高，深層乾旱 → 澆水無效
  - 夏季日蒸散大，澆水時機不對 → 蒸散損失 15-20%

總水利用效率：只有 50-70%
```

---

### 解決方案：IoT 灌溉的優勢

#### 優勢 1：自動化節省勞力
```
IoT 灌溉工作流程：
  - 感測器 24/7 監測土壤濕度、EC 值、溫度
  - 農民設定目標值（如土壤含水量 60%）
  - 系統自動調整灌溉（無需人工干預）
  - 農民只需每週檢查一次數據

節省成本：
  - 每年減少 150-300 小時勞動力
  - 折合年省 15-35 萬元
```

#### 優勢 2：產量穩定提升
```
農友案例（實戰數據）：

案例 1：某設施蔬菜農友（0.3 ha）
  導入前：設施蔬菜年產 60 噸（產量波動 ±15%）
  導入智能水肥系統：年產 87 噸（+ 45%）
  提升原因：灌溉精準度從 70% 提升到 95%

案例 2：某小規模瓜菜農友（0.3 ha）
  導入前：洋香瓜每季產量 15 噸
  導入感測灌溉：每季產量 17.5 噸（+ 16%）
  提升原因：減少過度灌溉導致的病害
```

#### 優勢 3：節水節肥 30-40%
```
IoT 灌溉如何節水：
  1. 精準灌溉：只澆需要的量，無浪費
  2. 時機最佳：根據土壤實時數據決定何時澆
  3. 蒸散監測：下雨時自動暫停灌溉

水利用效率：從 70% 提升到 90-95%

實際案例：
  0.25 ha 溫室，傳統灌溉年用水 150 m³
  導入 IoT 灌溉後，年用水 100 m³
  節省 50 m³ = 年省 5,000 元水費 + 環保
```

## 第 2 章：IoT 灌溉系統的核心組件

### 2.1 完整系統架構

```
【IoT 灌溉系統的 5 層架構】

第 1 層：感測層
  └─ 土壤濕度感測器（檢測土壤含水量）
  └─ 土壤 EC 值感測器（檢測肥料濃度）
  └─ 溫度感測器（檢測環境溫度）
  └─ 雨量感測器（偵測降雨，避免重複澆水）

第 2 層：控制層
  └─ 主控制器（決策中樞）
  └─ 水肥一體機（自動混肥並灌溉）
  └─ 定時器（備用機械控制）

第 3 層：執行層
  └─ 灌溉泵（抽水、加壓）
  └─ 肥料罐（儲存液肥或固體肥）
  └─ 滴灌帶（精準供水到植株根部）

第 4 層：通訊層
  └─ Wi-Fi 或 4G 模組（連接互聯網）
  └─ 雲端伺服器（數據儲存、分析）

第 5 層：應用層
  └─ 手機 App（農民可隨時監控）
  └─ 網頁後台（查看歷史數據、調整參數）
```

### 2.2 各組件的選購指南

#### 感測器選擇

| 感測器類型 | 功能 | 單價 | 優點 | 缺點 |
|----------|------|------|------|------|
| 土壤濕度（容量式） | 測土壤含水量 % | 800-2,000 元 | 精度好、壽命長 | 需定期校準 |
| 土壤 EC（電導率） | 測肥料濃度 | 1,500-3,000 元 | 可自動施肥 | 成本較高 |
| 溫度感測器 | 測環境溫度 | 300-800 元 | 便宜、易用 | 精度一般 |
| 雨量計 | 偵測降雨量 | 2,000-5,000 元 | 防止浪費 | 需定期清潔 |

**建議配置（小農標準）**：
- 土壤濕度感測器：3-5 個（分布在不同區域）
- 溫度感測器：1-2 個
- 雨量計：1 個（可選）
- **預算**：3,000-12,000 元

#### 控制系統選擇

| 控制方案 | 功能 | 投資 | 學習曲線 | 適用規模 |
|--------|------|------|---------|--------|
| 機械定時器（簡易） | 定時澆水，無智能 | 500-2,000 元 | 非常簡單 | < 0.1 ha |
| 單機智能控制器 | 根據濕度自動澆水 | 8,000-15,000 元 | 簡單 | 0.1-0.3 ha |
| 雲端智能系統 | 遠端監控 + 自動調整 | 15,000-50,000 元 | 中等 | 0.3-1 ha |
| 完整水肥一體系統 | 灌溉 + 施肥自動化 | 50,000-150,000 元 | 需專業維護 | > 1 ha |

**建議方案（小農）**：
- 初期：單機控制器（8,000-15,000 元）
- 後期升級：加裝雲端模組（5,000-10,000 元）

#### 灌溉硬體選擇

| 組件 | 規格 | 單價 | 備註 |
|------|------|------|------|
| 農業灌溉泵 | 0.5-1.5 HP | 3,000-8,000 元 | 根據面積選擇 |
| 肥料罐 | 100-500 L | 1,500-5,000 元 | 越大越省時 |
| 滴灌帶 | 16-20 mm | 15-30 元 / m | 覆蓋全面積 |
| 過濾器 | 200-400 微米 | 2,000-5,000 元 | 防止堵塞 |

## 第 3 章：小農 IoT 灌溉的三步導入法

### 步驟 1：從「機械定時器」開始（投資 2,000-5,000 元）

適合：極小規模（< 0.1 ha）或預算緊張的農友

#### 工作原理
```
定時器設定澆水時間（如：每天早 6 點澆 20 分鐘）
  → 時間到 → 打開電磁閥 → 灌溉泵啟動 → 澆水
  → 時間結束 → 自動關閉 → 完成

優點：
  ✅ 投資少（最便宜的自動化）
  ✅ 不需要電源以外的基礎設施
  ✅ 維護簡單

缺點：
  ❌ 無法根據天氣調整（下雨仍澆水，浪費）
  ❌ 無法根據土壤濕度調整（可能過乾或過濕）
  ❌ 無遠端控制（出差時無法改變澆水時間）
```

#### 系統配置
```
供水源 → 灌溉泵 → 定時器 → 主管 → 滴灌帶 → 植株

成本估算：
  - 灌溉泵（0.5 HP）：5,000 元
  - 定時器：1,500 元
  - 主管、滴灌帶、配件：2,000 元
  總投資：8,500 元

年運作成本：
  - 電費（泵 × 2 小時 / 天）：3,000 元 / 年
  - 維護、零件更換：1,500 元 / 年
```

### 步驟 2：升級到「簡易土壤濕度控制」（投資 12,000-20,000 元）

適合：小規模蔬果農友（0.1-0.3 ha）

#### 工作原理
```
土壤濕度感測器 → 主控制器 → 判斷是否澆水
  
判斷邏輯：
  IF 土壤含水量 < 40% → 啟動灌溉泵 → 澆水
  IF 土壤含水量 > 65% → 關閉灌溉泵 → 停止澆水
  ELSE → 保持當前狀態

優點：
  ✅ 根據實際土壤濕度調整（不過乾或過濕）
  ✅ 省水 20-30%（減少無謂澆水）
  ✅ 手機可遠端查看（需加 Wi-Fi 模組，另購）

缺點：
  ❌ 初投資較高
  ❌ 感測器需定期維護
  ❌ 無法自動調節肥料濃度
```

#### 系統配置
```
土壤濕度感測器 → 主控制器（WiFi） → 灌溉泵 → 滴灌帶 → 植株
                                    ↓
                           手機 App（監控）

成本估算：
  - 土壤濕度感測器（3 個）：4,500 元
  - 主控制器（WiFi）：10,000 元
  - 灌溉硬體（泵、管、配件）：5,000 元
  總投資：19,500 元

年運作成本：
  - 電費：2,500 元 / 年（感測器功耗低）
  - 維護、感測器校準：2,000 元 / 年
```

#### 實施案例：小規模設施農友
```
農友背景：
  - 設施面積 0.3 ha（3,000 m²）
  - 種植洋香瓜和小黃瓜輪作
  - 原有滴灌管，但靠手工澆水

導入過程：
  第 1 週：安裝 3 個濕度感測器（田間各區域 1 個）
  第 2 週：安裝主控制器和 Wi-Fi 模組
  第 3 週：調試，設定目標濕度 50-65%
  第 4 週：正式運作

成效（3 個月後）：
  - 澆水次數減少 30%（原先每天澆 2 次，現在 1.2 次）
  - 用水減少 25%
  - 病害發生率降低（因過度澆水減少）
  - 農民省下每天 30 分鐘的巡田時間
  
ROI 計算：
  投資 19,500 元
  年省水費：2,500 元
  年省勞力（30 min × 365 天 ÷ 60）：約 180 小時 = 18,000 元
  年節省小計：20,500 元
  投資回本時間：< 1 年
```

### 步驟 3：進階「智能水肥一體系統」（投資 50,000-150,000 元）

適合：中等規模蔬果農友（0.3-1 ha）、希望自動化程度最高

#### 工作原理
```
感測層：土壤濕度 + EC 值 + 溫度
    ↓
控制層：AI 引擎分析（土壤狀態 + 植株需求）
    ↓
執行層：同時自動調整
  - 水的量（多少升）
  - 肥的成分（N:P:K 比例）
  - 灌溉頻率（多久澆一次）
    ↓
結果：精準供應植株需要的「養分水」

優點：
  ✅ 灌溉 + 施肥完全自動化
  ✅ 可自動調節養液配方（根據植株生長階段）
  ✅ 產量提升最明顯（參考農友實績 +45%）
  ✅ 遠端完全監控

缺點：
  ❌ 投資大（50-150 萬元）
  ❌ 系統複雜，需定期維護
  ❌ 依賴廠商技術支援
```

## 第 5 章：常見問題與故障排除

### Q1：感測器埋在土裡，會不會損壞？

**答**：正常使用壽命 3-5 年。但需要定期維護：

```
預防措施：
  ✓ 選擇防水等級 IP67+ 的感測器
  ✓ 每季度檢查一次（挖出、擦乾、重新埋入）
  ✓ 每年進行校準（對比標準樣本）

故障表現及解決：
  現象：數值異常波動（如突然跳 20%）
  原因：感測器被土壤包裹、無法接觸真實濕度
  解決：挖出、清潔、重新埋入（稍淺一些）

  現象：數值停滯（一直顯示同一數值）
  原因：感測器故障、或連接線路斷裂
  解決：換新感測器、檢查線路
```

### Q2：系統停電會怎樣？

**答**：根據系統類型：

```
簡易定時器：
  → 停電期間無法工作
  → 恢復電力後，自動恢復運作
  → 無數據丟失風險

智能控制器（帶 UPS 備用電源）：
  → 停電 < 4 小時：照常運作（靠備用電池）
  → 停電 > 4 小時：系統暫停（等待恢復）
  → 恢復電力後，根據感測器數據自動調整
  
建議：
  ✓ 加配 UPS 不斷電系統（投資 5,000-10,000 元）
  ✓ 設定預警（停電超過 X 小時時，發手機通知）
```

### Q3：如果作物需要「催生」（突然加肥、加水），系統能配合嗎？

**答**：可以，有手動override模式：

```
情況 1：植株生長緩慢，想立即加肥
  → 農民手機 App → 點擊「手動灌溉」
  → 系統立即執行一次灌溉 + 加肥
  → 回到自動模式

情況 2：想暫停自動運作，進行人工管理
  → 在 App 設置「手動模式」
  → 系統暫停自動執行（但持續記錄數據）
  → 農民手工澆水期間，系統給予建議
  → 準備好後，切換回「自動模式」

情況 3：天氣預報說下雨，想暫停澆水
  → 手機 App → 設置「雨日禁止澆水」
  → 下雨期間系統暫停（但感測器繼續監測）
  → 雨停後，自動恢復
```

### Q4：投資這麼多，真的會回本嗎？

**答**：依農友規模和通路而定。參考實際案例：

```
小規模農友（0.3 ha）- 建議方案 2
  投資：19,500 元
  年省水費：2,500 元
  年省勞力：15,000-20,000 元（根據工資水準）
  年省肥料：3,000-5,000 元（減少無謂施肥）
  小計省資：20,500-27,500 元/年
  回本時間：0.7-1 年

中等規模農友（0.5 ha）- 建議方案 3
  投資：80,000 元
  年産量提升：+30-45%
  年増收入：50,000-80,000 元（假設銷售通路好）
  年省成本：15,000-25,000 元（省水、省肥、省工）
  小計年淨益：65,000-105,000 元
  回本時間：0.8-1.2 年
  
  注意：回本時間取決於「銷售通路」
    - 若產品只能走批發市場（時價波動大）→ 回本慢
    - 若産品能走直銷（單價穩定）→ 回本快
```

## 第 6 章：維護與進階優化

### 日常維護檢查清單

```
【每週檢查】
□ 感測器數據是否正常（無異常跳動）
□ 灌溉泵是否正常運作（無異常聲音）
□ 滴灌帶是否有堵塞（檢查出水是否均勻）
□ 手機 App 是否有警告訊息

【每月檢查】
□ 肥料罐液位是否足夠（不足時補充）
□ 過濾器是否堵塞（檢查出水壓力）
□ 線路是否有露水、鏽蝕（保持乾燥）
□ 感測器周邊是否有雜草遮擋（清除）

【每季檢查】
□ 感測器精度校準（對比標準樣本）
□ 灌溉泵是否有洩漏
□ 電磁閥是否反應靈敏
□ 整體系統數據分析（對比上季）

【每年檢查】
□ 感測器壽命是否接近（通常 3-5 年更換一次）
□ 軟體更新（廠商是否有新版本）
□ 完整系統評估（産量、成本、ROI）
```

### 進階優化：從「灌溉自動化」到「全面智能農場」

```
Level 1：單獨的灌溉系統 ✅ 你現在的位置
  - 感測 → 灌溉的自動化
  - 年省工、省水、産量 +10-15%

Level 2：灌溉 + 環控整合
  - 灌溉系統 + 溫度、濕度、CO₂ 監控
  - 環控系統根據灌溉狀態自動調整通風
  - 産量 +15-25%
  - 投資 +20,000-50,000 元

Level 3：灌溉 + 環控 + 無人機病蟲害監測
  - 定期無人機巡田（每週拍照）
  - AI 識別病蟲害早期跡象
  - 预防性用藥（減少病害損失）
  - 産量 +20-35%
  - 投資 +50,000-100,000 元

Level 4：完整智能農場
  - 灌溉 + 環控 + 病蟲害監測 + 銷售數據整合
  - 系統根據「市場需求」和「田間狀態」自動優化
  - 産量 +30-50%，利潤 +40-60%
  - 投資 > 150,000 元
```

## 第 7 章：農友導入案例匯總

### 案例 1：小農快速上手（方案 2）

```
農友背景：
  - 設施面積：0.3 ha（卡蜜拉洋香瓜 + 小黃瓜）
  - 原有條件：有滴灌管、有感測設備但未使用
  - 目標：自動澆水、減少勞力、提升産量

導入過程：
  月 1：購置主控制器 (WiFi) + 5 個土壤濕度感測器
  月 2-3：安裝、調試
  月 4：正式運作

成效（3 個月）：
  - 澆水次數減少 30%（從 2 次 → 1.2 次）
  - 病害減少（過度澆水導致的根腐病消失）
  - 産量提升 12% → 從年 72 萬增到 80 萬
  - 每週節省 3-4 小時巡田時間
  
投資回報：
  投資：19,500 元
  年增益：8 萬元（増産） + 2 萬元（省工省水）
  回本時間：< 1 年
```

### 案例 2：設施農場的智能升級（方案 3）

```
農友背景：
  - 設施面積：0.3 ha（番茄 + 小黃瓜輪作）
  - 原有條件：基礎灌溉但不穩定
  - 目標：灌溉 + 施肥完全自動化、提升産量 40% 以上

導入過程：
  第 1 月：簽約廠商、採購智能水肥一體系統
  第 2-3 月：安裝、調試、參加廠商培訓
  第 4 月：正式運作

系統內容：
  - 智能水肥機：可自動配製 5 種肥料配方
  - 10 個多參數感測器（濕度、EC、溫度）
  - 環控整合（溫度、濕度聯動）
  - 雲端數據平台 + 手機 App

成效（第一年）：
  - 産量從 60 噸 → 87 噸（+45%）
  - 用水減少 40%（從 150 m³ → 90 m³）
  - 肥料成本減 30%（精準施肥）
  - 農民勞動時間減 60%（大部分自動化）
  
投資回報：
  投資：80,000 元
  年增益：
    - 産量增加 27 噸，假設批發 8,000 元 / 噸
    - 增収 216,000 元（保守估計）
    - 省水 60 m³ × 100 元 = 6,000 元
    - 省肥料：24,000 元
    - 省勞力（年 300 小時 × 150 元 / 小時）：45,000 元
  合計年增益：291,000 元
  回本時間：3 個月
  第 2 年起，年利潤 291,000 元皆為淨益
```

### 選購建議

```
詢問廠商時的 5 個關鍵問題：

1. 「系統使用壽命多長？感測器、控制器各需要多久更換一次？」
   ✓ 好答案：感測器 3-5 年、控制器 5-7 年、終身免費軟體更新

2. 「如果系統故障，維修期間多久？緊急情況能否上門？」
   ✓ 好答案：24 小時內回應、嘉義地區 2 小時內上門

3. 「手機 App 資料會不會丟失？伺服器在哪？」
   ✓ 好答案：雲端備份、台灣伺服器、用戶可隨時下載資料

4. 「有沒有其他嘉義農友用過？能否介紹？」
   ✓ 好答案：能提供至少 3-5 位在地使用案例、可親自拜訪

5. 「後續如果想升級（如加感測器、加環控），成本多少？」
   ✓ 好答案：組件化設計、可逐步擴展、不綁定廠商
```
